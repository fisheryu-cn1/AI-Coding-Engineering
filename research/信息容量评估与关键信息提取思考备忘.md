**信息容量评估与关键信息提取 — 备忘文档**

---

### 一、核心概念区分

| 概念 | 定义 | 单调性 | 适用场景 |
|------|------|--------|----------|
| **信息容量（Information Capacity）** | 文档中不可压缩的有效信息总量，绝对量 | ✅ 严格单调 | 评估文档"干货"总量，信息多则得分高 |
| **信息密度（Information Density）** | 单位长度内的信息量，相对量 | ❌ 非单调 | 评估表达效率，受长度归一化影响 |

**关键结论**：若要求"信息容量大的文档得分一定更高"，必须评估**绝对信息量**，放弃任何密度/归一化指标。

---

### 二、评估方法：累积描述长度（CDL）

**理论基础**：将大模型视为近似通用概率估计器，累加文档逐 token 的条件惊讶值（surprisal）。

$$ \text{CDL}(D) = -\sum_{i=1}^{n} \log_2 P_M(t_i | t_1...t_{i-1}) \quad \text{(bits)} $$

**单调性保证**：
- 追加可预测的冗余 token → 贡献 ≈ 0，总分几乎不变
- 追加不可预测的新信息 token → 贡献 > 0，总分严格增加

**因此 CDL 对有效信息规模具备近似严格单调性。**

### 三、工程实现路径（本地 Qwen）

#### 推荐配置
- **模型**：Qwen2.5-1.5B/3B/7B-Instruct（4-bit 量化，消费级显存可跑）
- **核心依赖**：`transformers`, `torch`, 可选 `jieba`（中文分词聚合）

#### 关键实现要点
1. **逐 token 增量推理**：使用 KV-Cache，避免重复计算，将复杂度从 O(n²) 降至 O(n)
2. **精确 logprob 提取**：本地模型可获取完整 logits 向量，精确提取任意 token 的负对数概率（优于 API 的有限 logprobs）
3. **超长文档处理**：采用分层递归评估
   - 分段计算本地 CDL
   - 计算条件 CDL（基于前文摘要作为前缀）
   - 增量信息 = 条件 CDL，避免跨段重复计算

#### 标准化与可比性
- 固定单一模型作为"标准概率估计器"，保证内部可比
- 不跨模型比较绝对 CDL 值（不同模型概率分布不同）
- 可建立自校准基线：用模型在标准语料上的平均 token 熵作为参考点

### 四、同步提取关键信息词汇

在计算 CDL 的同一轮前向传播中，零额外成本提取前 n 个关键信息词汇。

**原理**：每个 token 的 surprisal \( s_i = -\log_2 P(t_i|t_{<i}) \) 即其信息贡献量。高 surprisal 的 token 是模型难以预测的"信息锚点"。

#### 提取流程
1. **逐 token 计算 surprisal**（与 CDL 同次推理）
2. **Token → 词汇聚合**：
   - 中文：按字计算后，用 jieba 或 tokenizer 词边界聚合
   - 聚合方式：求和（总贡献）、平均（单位贡献）、最大（峰值捕捉）
3. **去噪过滤**：
   - 排除孤立极端值（>15 bits 的孤立 token，视为乱码/噪声）
   - 停用词惩罚降级
   - 要求局部上下文具备一定连贯性（非孤立噪声）

#### 进阶：边际信息贡献（更精准）
对候选词做二次验证：删除该词后重算局部 CDL 变化 \( \Delta I_w = L_{\text{masked}} - L_{\text{full}} \)。
- \( \Delta I_w > 0 \)：该词承载不可压缩信息
- \( \Delta I_w \approx 0 \)：该词冗余

**工程优化**：先用 surprisal 排序筛选候选集（前 5%），再对候选集做边际贡献精排，降低 90% 计算量。

### 六、局限与边界

| 问题 | 影响 | 缓解策略 |
|------|------|----------|
| **训练数据污染** | 对模型训练集内文档概率高估，CDL 低估 | 使用较新的本地模型，或引入未知性惩罚 |
| **模型规模偏见** | 小模型（1.5B/3B）对罕见专业术语可能过度惊讶 | 特定领域可用少量语料做 LoRA 微调校准 |
| **绝对值不可比** | 不同模型/tokenizer 的 CDL 绝对值差异大 | 固定评估器，仅内部相对比较 |
| **计算成本** | 精确 CDL 需 O(n) 次推理 | KV-Cache + 分层采样 + 边际贡献精排候选集 |
| **超长上下文** | 超出模型窗口后长程依赖丢失 | 分层递归：局部 CDL + 全局条件去重 |

### 七、最小可行方案（MVP）

1. **环境**：部署 Qwen2.5-3B-Instruct（4-bit 量化）
2. **输入**：待评估文档
3. **输出**：
   - `total_bits`：文档总信息容量（严格单调指标）
   - `top_n_keywords`：按聚合 surprisal 排序的前 n 个关键信息词汇
   - `redundancy_ratio`：条件 CDL / 本地 CDL，反映文档内部重复度
4. **验证**：向文档追加已知内容，观察新增实体词 surprisal 显著高于重复词，且总 CDL 增加量等于新增 token surprisal 之和。
